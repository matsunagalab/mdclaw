# MDAgentBench Integrity Policy Rollout

The v1.0.x scorer ships every artifact integrity check in **warn** mode:
a missing citation, an undersized evidence report, or a template-stub
`methods.md` records an `integrity_warnings` entry and applies the existing
`-0.05/check (capped -0.2)` weighted_total penalty, but it does not clamp
the score to zero.

This document describes the criteria, order, and timeline for promoting
each task to **reject** mode, where any artifact integrity warning drops
`weighted_total` to 0 and forces `status="failed"` regardless of the
ground-truth match.

## Why two phases

v1.0.x went live with new integrity surface (file-size floors, PNG magic,
citation-pool anchors, template-marker detection). Submissions that were
honest under v0.1 — for example, an external agent that always emitted
`methods.md` but skipped `evidence_report.evidence.md_metrics` because it
was previously optional — would suddenly score zero under a hard reject
without a grace window. Warn mode gives those agents a few release
cycles to migrate while still penalizing fabrication.

## Criteria for promoting a task to reject

A task moves to `integrity_policy="reject"` only after **all** of the
following hold in the v1.0.x telemetry:

1. **No false positives across two consecutive releases.** The integrity
   warnings recorded for that task on the dataset's reference honest
   submission (e.g. `tests/fixtures/benchmark/fake_submissions.py` honest mode
   for the curator side, and at least one external runner for the
   ecosystem side) must be zero.
2. **At least one observed real-world fabrication detected.** Either a
   `tests/fixtures/benchmark/*` regression fixture, a logged Haiku-class
   run, or a self-test scenario must demonstrate that a fabricated
   submission for the same task triggers >= 2 integrity warnings (i.e.
   the artifact layer is actually catching the failure mode, not just
   noisy on edge cases).
3. **Documented in the changelog.** Each promotion lands in a release
   note so external runners know to revalidate.

## Rollout order

Promote in increasing order of risk to legitimate external submissions:

1. **T08** (figures): PNG magic + byte floors are unambiguous. Real
   submissions either render a figure or they don't; there is no honest
   middle ground that would be penalized.
2. **T02** (guardrail): Already required structured refusal; tightening
   evidence_report bytes from warn to reject only catches "completed
   refusal with empty explanation," which is itself dishonest.
3. **T06, T07** (scientific answer): MD-evidence requirement is the
   point. After two release cycles where honest external agents either
   produce `evidence.md_metrics` or block, promote.
4. **T09** (study methods): Markdown structure + min_bytes are easier to
   game with a verbose template than the others, so this is the riskiest
   to promote — wait until external submissions consistently provide
   real `## Methods` / `## Limitations` sections.
5. **T01, T03, T04, T05**: Promote only after adding their integrity
   blocks (see "Adding integrity to execution tasks" below) and waiting
   one release cycle.

## Adding integrity to execution tasks

T01, T03, T04, T05 currently lean on deterministic `trajectory_rescan` /
`rmsd_recompute` / `topology_solvent_rescan` for end-to-end correctness.
The artifact integrity surface is intentionally narrower for these tasks
because the deterministic layer already requires loading the physical
files. However:

- `manifest.status="completed"` + empty `manifest.outputs.trajectories`
  is a partial-fabrication path that the deterministic layer catches
  only via `manifest_metrics_consistency` (existing v0.1 check), which
  fires only when `metrics.execution.completed=true`. An honest-looking
  manifest without `metrics.execution` can slip past.
- `evidence_report.json` left as the 52-byte template (observed in
  Haiku v3 for T09 and in Haiku v3/v4 for T01/T02/T08) is silent under
  the deterministic layer.

v1.0.x adds the following integrity_checks to T01/T04/T05:

- `evidence_report_real_bytes` (artifact_min_bytes, 200 byte floor)
- `evidence_report_not_template` (template_markers covering
  `"Generated by create_benchmark_submission_template"`,
  `"fill task-specific"`, `"Replace before scoring"`)
- `completed_requires_trajectory` (status_artifact_floor: when
  `manifest.status="completed"`, require a non-empty trajectory file
  listed under `manifest.outputs.trajectories`)

T03 receives the same `evidence_report_real_bytes` /
`evidence_report_not_template` pair plus a `prepared_structure_real_bytes`
floor (5 KB) when status="completed".

These ship in `warn` mode and follow the rollout order above.

## Rollback procedure

A task in `reject` mode can be reverted to `warn` by editing
`benchmarks/mdagentbench/tasks/<task_id>/task.json` and changing
`scoring.integrity_policy` from `"reject"` to `"warn"`. The scoring code
treats both values uniformly except for the final clamp, so no other
code change is needed.

Cross-run records (`benchmark_runs/runs.jsonl`,
`benchmark_runs/summaries.jsonl`) are NOT retroactively updated; any
previously-rejected submissions remain at `weighted_total=0` in their
score.json until rescored.

## Promotion history

- **v1.1 (2026-05-11): T02 + T08 promoted to reject.** Honest
  `tests/fixtures/benchmark/fake_submissions.py` fixtures and the Haiku v3/v4 telemetry showed
  zero false positives on these tasks; the regression tests
  `test_reject_policy_zeros_silent_refusal_t02_submission` and
  `test_reject_policy_zeros_text_disguised_as_png_t08` pin the new
  behavior (50-byte refusal explanation or text-as-PNG figure now
  clamps weighted_total to 0).
- v1.2 (planned): promote T06 + T07 once a second honest-run cycle
  confirms no false positives on the citation_pool / md_metrics
  checks. Currently a blocked T06 generates several integrity
  warnings, which is correct (a blocked submission is at 0 already)
  but should be documented in release notes so external runners
  do not over-react.
- v1.3+ (planned): T09, then T01/T03/T04/T05 in lockstep.

These dates are not contractual; promotion follows the criteria, not
the calendar.
